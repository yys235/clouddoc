## 1. 范围说明

这份文档只基于视频中可以明确观察到的界面和交互来整理功能点。

结论分两类：

- `明确识别`：视频里可以直接看到的功能、菜单项、交互结果
- `合理推断`：从画面结构和交互结果推断出的实现要求

没有在视频中稳定出现的能力，不在这里写成强需求。

## 2. 视频内容概览

视频时长约 `41.1s`，画面分为两部分：

- 上方是流程图/节点画布
- 下方是一个与画布共存的块编辑区域

视频主要演示的不是整页文档编辑，而是：

- 在一个带流程图画布的页面中
- 对下方文本块进行结构化编辑
- 通过 `/` 命令菜单插入或切换块类型

这说明目标产品不是简单富文本输入框，而是：

- `画布 + 块文档` 的混合编辑场景
- 块编辑区需要服务于流程图/节点说明/备注内容

## 3. 明确识别到的功能点

### 3.1 页面结构

明确识别：

- 页面上方存在流程图/节点画布
- 页面下方存在独立的块编辑区域
- 块编辑区域支持多行、多块内容

研发结论：

- 页面需要支持混合布局
- 文本块不能只是普通评论框，而应是结构化内容区

### 3.2 块编辑基础能力

明确识别：

- 存在独立文本块
- 不同块有不同的视觉样式
- 光标可进入块内编辑
- 块与块之间有垂直排列关系

研发结论：

- 编辑器底层必须是块模型
- 至少支持块级新增、切换类型、排序、删除

### 3.3 左侧块手柄

明确识别：

- 块左侧有一个小型操作手柄
- 手柄上有类型图标/标识
- 手柄旁有加号入口

合理推断：

- 左侧手柄是块级主操作入口
- 手柄还应承担拖拽排序能力

研发要求：

- 每个块左侧有统一手柄区
- 手柄 hover 显示，选中块时高亮
- 手柄支持打开块菜单
- 手柄支持拖拽排序

### 3.4 `/` 命令菜单

明确识别：

- 在空块或块起始处输入 `/`
- 会弹出块命令菜单
- 菜单支持分类展示
- 菜单会随着输入内容过滤

研发要求：

- slash menu 必须支持关键字过滤
- 菜单需支持键盘选择和回车确认
- 菜单位置应贴近当前编辑块

### 3.5 块类型菜单项

视频中能明确看到的菜单项包括：

- `文本`
- `一级标题`
- `二级标题`
- `三级标题`
- `有序列表`
- `无序列表`
- `代码块`
- `引用`
- `分割线`
- `链接`
- `任务`
- `图片`

研发结论：

- 这些至少应作为第一批块类型支持项

### 3.6 标题块

明确识别：

- 标题块比正文更大更粗
- 标题块前出现了类似折叠箭头/展开标识

合理推断：

- 标题块可能支持折叠下级内容
- 至少要预留标题层级结构

研发要求：

- 支持 H1/H2/H3
- 标题块样式明显区别于正文
- 数据模型保留折叠字段

### 3.7 列表与任务块

明确识别：

- 菜单中出现 `有序列表`、`无序列表`、`任务`

研发要求：

- 列表块支持多行条目
- 任务块支持未完成/已完成状态
- 任务块应支持键盘连续输入

### 3.8 代码块、引用、分割线、链接

明确识别：

- 菜单中出现 `代码块`、`引用`、`分割线`、`链接`

研发要求：

- 代码块需有独立样式和多行输入能力
- 引用块需有左侧强调线或背景差异
- 分割线是无文本内容块
- 链接块至少支持 URL 文本和展示文本

### 3.9 图片块

明确识别：

- 菜单中出现 `图片`

研发要求：

- 需支持插入图片块
- 至少支持上传或粘贴图片
- 图片块应支持宽度约束和删除

## 4. 交互流程拆解

### 4.1 新建块流程

视频中可推断的流程：

1. 用户在空白行或块尾输入 `/`
2. 系统弹出命令菜单
3. 用户选择块类型
4. 当前块切换为目标块类型
5. 用户继续输入内容

### 4.2 标题块编辑流程

视频中可推断的流程：

1. 输入 `/一级标题` 或从菜单选择标题
2. 块切换为标题样式
3. 输入标题文字
4. 标题可能进入文档结构树或折叠结构

### 4.3 拖拽排序流程

合理推断：

1. 鼠标 hover 左侧手柄
2. 按住手柄拖动
3. 目标位置出现插入线/占位态
4. 释放后完成块排序

### 4.4 任务块流程

合理推断：

1. 使用 `/任务`
2. 生成任务块
3. 输入任务内容
4. 勾选完成状态

## 6. UI 设计要求

### 6.1 块区域

- 内容区保持连续文档感
- 控件要弱化，不能像后台表单
- 左侧手柄是主入口
- hover 时显示手柄和加号

### 6.2 命令菜单

- 菜单宽度中等，不要全屏
- 顶部支持快捷类型切换
- 中部显示命令结果
- 菜单项需有图标、标题、说明
- 支持分类标题，例如 `基础`、`常用`

### 6.3 标题视觉

- 一级标题尺寸最大
- 二级标题次之
- 三级标题再收一档
- 标题与正文有明显层级

### 6.4 分割线与引用

- 分割线块需要轻量、低干扰
- 引用块要用边线或底色强调

## 7. 键盘交互要求

- `Enter`：在当前块下方创建新块
- `Shift+Enter`：块内换行
- `ArrowUp/Down`：块边界跨块移动
- `ArrowLeft/Right`：行首/行尾跨块移动
- `/`：打开命令菜单
- `Enter`：在命令菜单中确认当前命令
- `Esc`：关闭命令菜单

## 8. 数据模型建议

建议块模型至少包含以下字段：

```ts
type BlockType =
  | "paragraph"
  | "heading_1"
  | "heading_2"
  | "heading_3"
  | "ordered_list"
  | "bullet_list"
  | "task"
  | "code_block"
  | "quote"
  | "divider"
  | "link"
  | "image";

type Block = {
  id: string;
  type: BlockType;
  text?: string;
  attrs?: Record<string, unknown>;
  children?: Block[];
};
```

不同块建议的 attrs：

- `task`: `checked`
- `heading_*`: `collapsed`
- `code_block`: `language`
- `link`: `href`, `title`
- `image`: `src`, `width`, `height`, `alt`

## 9. 前端实现建议

- 编辑器核心：`ProseMirror / TipTap` 或自研轻块编辑层
- 块菜单：独立浮层组件
- 手柄：独立块装饰组件
- 拖拽：优先 HTML5 DnD 或 `dnd-kit`
- slash menu：按输入上下文触发

组件拆分建议：

- `BlockEditor`
- `BlockRow`
- `BlockHandle`
- `BlockMenu`
- `SlashMenu`
- `BlockRenderer`
