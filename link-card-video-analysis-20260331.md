# 链接卡片功能拆解（基于 20260331092817_rec_.mp4）

## 1. 视频结论
该视频展示的是“粘贴链接后自动识别，并允许切换多种链接展示视图”的完整交互。核心不是单纯插入 URL，而是把 URL 识别为一个独立的内容块，并在块级工具栏中提供展示模式切换。

## 2. 识别到的主流程
1. 用户在空白文档块中粘贴一个 URL。
2. 系统立即把该 URL 识别为链接块，而不是普通正文。
3. 链接块生成后，块上方出现块级工具条。
4. 工具条里有“视图切换”入口，默认初始为“链接视图”。
5. 用户打开视图菜单，可切换为：
- 链接视图
- 标题视图
- 卡片视图
- 预览视图
6. 当切换为“预览视图”后，系统展示网页预览区域。
7. 预览内容加载完成后，块仍保留上方工具条，可继续切换回其它视图。

## 3. 明确识别到的功能点

### 3.1 链接自动识别
- 在普通空块中粘贴 URL，会自动转为链接块。
- 不是要求用户先手动切换块类型。
- 识别对象至少包括标准 `http/https` 链接。

### 3.2 链接块独立化
- 链接不是正文里的普通文本样式，而是独立块。
- 链接块拥有自己的块级工具栏。
- 链接块可被选中，具有独立边界和独立操作。

### 3.3 链接视图模式切换
视频里明确出现 4 种视图：
- `链接视图`
  - 主要显示 URL 文本本身。
- `标题视图`
  - 主要显示页面标题，不强调卡片容器。
- `卡片视图`
  - 使用更明显的卡片样式呈现链接信息。
- `预览视图`
  - 展示网页预览内容，接近嵌入浏览预览。

### 3.4 视图菜单交互
- 视图菜单由块工具栏中的下拉入口打开。
- 当前选中的视图项有勾选状态。
- 菜单切换后块立即重绘，无需再次确认。

### 3.5 网页预览加载
- 切到 `预览视图` 后，块内容区展示网页缩略预览。
- 视频表现看起来不是简单展示标题，而是有真实页面截图/嵌入内容。
- 预览区域支持继续被块级工具栏控制。

### 3.6 块级工具栏
从视频可识别到工具栏包含这些入口：
- 刷新/重新加载预览
- 视图切换下拉（当前显示“预览视图”等）
- 栅格/布局类入口
- 更多操作入口
- 评论入口

说明：后 3 项图标能确认存在，但视频不足以完全确认它们的具体功能定义。

### 3.7 选中态与块边框
- 预览块被选中时会出现浅蓝描边。
- 块左上角有块手柄/块类型入口。
- 工具栏悬浮于块上方，而不是占正文固定列。

## 4. 可推断的交互要求
以下是基于视频可合理推断、但视频未完全展示实现细节的部分。

### 4.1 URL 粘贴拦截
- 需要在输入层拦截粘贴事件。
- 如果粘贴内容是 URL，且当前块为空或当前选择满足规则，则把当前块转成链接块。

### 4.2 元数据抓取
为了支持 `标题视图 / 卡片视图 / 预览视图`，系统至少要有以下其中一种能力：
- 后端抓取网页 metadata（title、description、icon、image）
- 调第三方 link-preview 服务
- 对特定域名做 embed 适配

### 4.3 预览渲染策略
视频里的预览效果可能来自这几种方案之一：
- 后端截图后展示缩略图
- Open Graph 图片 + 标题描述
- 对支持嵌入的站点直接 iframe/embed

从视频表现看，更像“可视预览块”，不只是纯 OG 卡片。

## 5. 推荐的数据模型
建议把链接块从一开始就设计成结构化块，而不是简单字符串。

```json
{
  "type": "link_card",
  "attrs": {
    "url": "https://project.feishu.cn/home",
    "view": "preview",
    "title": "飞书项目",
    "description": "专业项目管理工具的第一选择",
    "site_name": "飞书项目",
    "icon": "https://.../favicon.ico",
    "image": "https://.../cover.png",
    "preview_url": "https://...",
    "embed_url": "https://...",
    "status": "ready"
  }
}
```

关键字段建议：
- `url`：原始链接
- `view`：`link | title | card | preview`
- `title`
- `description`
- `site_name`
- `icon`
- `image`
- `embed_url` 或 `preview_url`
- `status`：`loading | ready | error`

## 6. 前端功能拆分建议

### P0
- 粘贴 URL 自动识别
- 自动转链接块
- 链接视图 / 标题视图切换
- 链接 metadata 拉取
- 视图菜单和当前项勾选

### P1
- 卡片视图
- 预览视图
- 预览加载中和失败态
- 刷新预览

### P2
- 特定域名嵌入优化
- 评论/更多操作
- 链接块拖拽和复制增强
- 批量粘贴多链接处理

## 7. 建议的后端能力
- `POST /api/link-preview`
  - 输入：`url`
  - 输出：title、description、site_name、icon、image、preview strategy
- 需要超时、缓存、域名白名单/黑名单、抓取失败兜底。

## 8. 当前项目与视频能力差距
当前项目已有：
- 手动链接块类型
- `标题 | URL` 形式的链接块存储
- 阅读态链接卡片渲染基础

当前项目缺少：
- 粘贴 URL 自动识别
- 自动抓取标题/站点信息
- 视图切换（链接/标题/卡片/预览）
- 预览块
- 链接块工具栏的完整交互

## 9. 建议的实现顺序
1. 先做 URL 粘贴自动识别
2. 再做后端 metadata 抓取接口
3. 再做 `链接视图 / 标题视图 / 卡片视图`
4. 最后再做 `预览视图`

原因：
- 前 3 步能先把“像飞书”的核心交互做出来
- `预览视图` 的成本最高，适合放在后面
