# 文档权限与分享专项需求文档

## 1. 背景

当前 CloudDoc 已具备基础文档编辑、评论、通知和用户系统能力，但“文档可见性、分享访问控制、分享页隔离、分享安全边界”还没有形成完整闭环。

本次目标是建立一套明确、可实现、可测试的文档权限与分享体系，满足以下核心要求：

- 私有文档默认只有作者或被授权用户可访问
- 公开文档所有人都可访问
- 私有文档可生成独立分享链接
- 分享链接可配置有效期和访问密码
- 分享链接必须使用独立地址，不暴露原文档编辑地址
- 分享页与普通文档页使用同一套前端渲染代码，保证显示效果一致
- 分享页必须强制只读，不能从分享界面进入编辑

## 2. 目标

### 2.1 产品目标

- 让用户可以明确控制文档对谁可见
- 让用户可以安全地把私有文档对外分享出去
- 保证分享访问体验与原文档阅读体验一致
- 保证分享访问不会泄露编辑能力

### 2.2 工程目标

- 不引入第二套阅读页实现
- 复用现有文档页组件与统一文本 surface
- 权限判断逻辑集中在后端
- 分享 token 独立于文档原始 ID 路由，不可通过前端拼接原文链接绕过

## 3. 术语定义

### 3.1 文档可见性

- `private`：私有文档，仅作者与显式授权用户可访问
- `public`：公开文档，任何人都可访问

### 3.2 分享链接

- 针对文档生成的独立访问入口
- 路径应类似：`/share/{token}` 或 `/s/{token}`
- 不等于原文档地址 `/docs/{docId}`

### 3.3 分享访问

- 通过 share token 访问文档
- 可受密码、有效期、状态控制
- 默认仅允许查看，不允许编辑

## 4. 需求汇总与扩展说明

基于你的原始要求，补充并扩展后的需求如下。

### 4.1 原始需求

- 文档需要区分权限
- 私有文档只能自己看
- 公开文档都可以访问
- 私有文档可分享出去
- 分享可带时间限制或无限期
- 分享可设置访问密码
- 分享文档必须生成新链接
- 分享查看页要使用同一套前端代码
- 分享界面必须锁死，不能编辑

### 4.2 补充约束

- 私有文档的分享能力不改变原文档自身权限；只是新增一条受控访问通道
- 文档作者应可随时：
  - 关闭分享
  - 修改分享密码
  - 修改过期时间
  - 重新生成分享链接
- 分享页只允许查看，不允许：
  - 进入编辑模式
  - 评论
  - 删除
  - 收藏
  - 上传图片
  - 打开块工具菜单
  - 创建分享中的二次分享
- 公开文档直接访问原链接即可，不强依赖分享链接
- 私有文档若未开启分享，则通过分享链接访问应拒绝

### 4.3 推荐扩展

为了避免后续返工，建议首版一起纳入：

- 分享状态：`active / disabled / expired`
- 分享访问审计字段：
  - 最近访问时间
  - 访问次数
- 支持“重新生成 token”
- 公开文档仍可额外生成分享链接，用于密码或有效期保护的外发场景

## 5. 权限模型

### 5.1 文档可见性字段

建议给文档新增统一可见性字段：

- `visibility`
  - `private`
  - `public`

默认值：

- 新建文档默认 `private`

### 5.2 访问规则

#### 私有文档

- 作者可访问
- 被显式授权的用户可访问
- 其他用户访问 `/docs/{docId}` 返回无权限
- 但若启用有效分享链接，则可通过 `/share/{token}` 访问

#### 公开文档

- 任意用户可通过 `/docs/{docId}` 访问
- 未登录用户也可访问

### 5.3 分享访问规则

分享访问优先看 share link 状态，不直接继承编辑权限。

规则顺序建议：

1. share token 是否存在
2. share 是否启用
3. 是否过期
4. 是否需要密码
5. 密码是否正确
6. 满足后返回只读访问

## 6. 分享模型

### 6.1 分享链接核心字段

已有 `share_links` 表，建议按以下语义使用/扩展：

- `id`
- `document_id`
- `token`
- `access_scope`
- `permission_level`
- `password_hash`
- `expires_at`
- `is_active`
- `created_by`
- `created_at`
- `updated_at`

### 6.2 字段建议

- `token`
  - 高熵随机串
  - 不使用可预测 ID
- `permission_level`
  - 首版仅允许 `view`
  - 即使表里有 `edit` 枚举，分享页也不要开放
- `password_hash`
  - 不存明文密码
  - 使用与用户密码一致的 hash 策略
- `expires_at`
  - `null` 表示长期有效
- `is_active`
  - 关闭分享时置为 `false`

## 7. 页面与交互设计

### 7.1 文档页权限入口

在文档页顶部增加“权限/分享”入口。

弹窗或抽屉内包含：

- 文档可见性
  - 私有
  - 公开
- 分享设置
  - 是否启用分享
  - 有效期
  - 是否启用密码
  - 设置/修改密码
  - 复制分享链接
  - 重新生成链接
  - 关闭分享

### 7.2 分享页路由

建议新增：

- `/share/[token]`

或者：

- `/s/[token]`

推荐短链形式：

- 更适合复制和传播

### 7.3 分享页显示要求

分享页必须复用当前文档查看组件，不允许维护第二套页面。

实现要求：

- 使用相同的 `DocumentPage` / 统一文本 surface / 图片块 / 链接块 / 评论高亮渲染逻辑
- 通过一个显式模式控制：
  - `accessMode = owner | collaborator | public_share`
- 当 `accessMode = public_share` 时：
  - 强制 `editable = false`
  - 隐藏一切编辑相关 UI

### 7.4 分享页必须隐藏的能力

- 编辑模式切换
- 自动保存提示
- 删除按钮
- 评论创建
- 评论回复
- 图片上传
- 块拖拽
- 块菜单
- 新建块
- 收藏
- 再分享

### 7.5 密码访问流程

访问分享链接时：

1. 后端判断该 token 是否需要密码
2. 若需要，则前端显示密码输入页
3. 验证成功后，设置短期访问凭证
4. 再进入只读分享页

访问凭证建议：

- `httpOnly cookie`
  - 绑定 share token
  - 过期时间不超过 share 本身有效期

## 8. 前端实现方案

### 8.1 页面复用原则

不要做单独的“分享版文档页”。

建议做法：

- 保持 `/docs/[docId]` 走当前文档页
- 新增 `/share/[token]`
- `/share/[token]` 解析分享后，构造一个“只读文档上下文”，然后复用同一组件

### 8.2 前端上下文设计

建议新增 `DocumentAccessContext` 或页面 props：

- `accessMode`
- `canView`
- `canEdit`
- `canComment`
- `canManage`
- `isSharedView`

其中分享页固定：

- `canView = true`
- `canEdit = false`
- `canComment = false`
- `canManage = false`
- `isSharedView = true`

### 8.3 组件控制原则

所有会导致修改数据的 UI 都必须统一受 `canEdit/canManage/canComment` 控制，而不是仅靠页面路由控制。

否则未来容易出现：

- 分享页虽然看似只读，但某个按钮漏判断仍可操作

## 9. 后端实现方案

### 9.1 文档访问接口

原文档接口：

- `GET /api/documents/{id}`

新增分享接口：

- `GET /api/share/{token}`
- `POST /api/share/{token}/verify-password`

返回内容建议：

- 文档元数据
- 文档内容
- 分享信息摘要
- 权限能力标记

### 9.2 分享管理接口

建议新增：

- `POST /api/documents/{id}/share-links`
  - 创建或更新分享
- `GET /api/documents/{id}/share-links`
  - 查看当前分享配置
- `PATCH /api/share-links/{id}`
  - 修改有效期、密码、状态
- `POST /api/share-links/{id}/rotate`
  - 重置 token
- `DELETE /api/share-links/{id}`
  - 关闭分享

### 9.3 权限判断原则

访问 `/docs/{docId}`：

- 按文档可见性 + 所有者 + 授权关系判断

访问 `/share/{token}`：

- 只按 share token 状态判断
- 不允许因此获得编辑权限

## 10. 数据模型调整建议

### 10.1 documents

新增字段：

- `visibility VARCHAR(16) NOT NULL DEFAULT 'private'`

### 10.2 share_links

确认/新增字段：

- `password_hash TEXT NULL`
- `is_active BOOLEAN NOT NULL DEFAULT true`
- `access_count INTEGER NOT NULL DEFAULT 0`
- `last_accessed_at TIMESTAMPTZ NULL`

## 11. 关键业务规则

### 11.1 私有文档

- 默认不可被未授权用户访问
- 复制原文档链接给别人无效

### 11.2 公开文档

- 直接访问原文档链接即可查看

### 11.3 分享链接

- 必须与原文档 URL 分离
- 关闭分享后原 token 立即失效
- 重新生成 token 后旧 token 立即失效
- 过期分享不可继续访问

### 11.4 分享页只读

任何情况下都不能通过分享页进入编辑。

这条需要同时在：

- 前端 UI
- 前端 API 调用
- 后端接口权限

三层锁死。

## 12. 风险与注意事项

### 12.1 只做前端隐藏不够

必须做后端权限拦截，否则直接调接口仍可能写数据。

### 12.2 分享密码不能明文存储

必须 hash。

### 12.3 分享页必须使用同一套组件

否则会重新引入“阅读模式和编辑模式显示不一致”的历史问题。

### 12.4 原文档链接和分享链接权限不能混淆

分享访问成功不等于拥有文档本体权限。

## 13. 开发计划

### Phase 1：权限模型与只读分享主链路

- 文档新增 `visibility`
- `/docs/{id}` 访问按权限拦截
- `share_links` 补齐字段
- 生成独立分享链接
- `/share/{token}` 只读查看
- 分享页复用同一套文档查看组件
- 后端禁止分享页写操作

交付标准：

- 私有文档原始链接无法被他人查看
- 公开文档原始链接可直接访问
- 分享链接可独立打开文档
- 分享页不能编辑

### Phase 2：密码与有效期

- 分享密码设置与验证
- 分享有效期控制
- token 失效提示页
- 关闭分享与重置 token

交付标准：

- 错误密码不能访问
- 过期链接不能访问
- 重置 token 后旧链接失效

### Phase 3：管理面与审计

- 权限/分享弹窗
- 分享状态查看
- 访问次数/最近访问时间
- 复制链接、修改密码、改有效期、关闭分享

交付标准：

- 分享配置可在 UI 完整管理
- 操作结果立即生效

## 14. 测试计划概要

### 14.1 权限测试

- 私有文档作者可访问
- 私有文档其他用户不可访问
- 公开文档所有人可访问

### 14.2 分享测试

- 分享链接可访问
- 原文链接仍按原权限
- 关闭分享后访问失败
- 过期后访问失败
- 密码错误访问失败

### 14.3 只读测试

- 分享页不可切编辑模式
- 分享页不可保存
- 分享页不可评论
- 分享页不可删除
- 分享页不可上传图片

### 14.4 一致性测试

- 同一文档在 `/docs/{id}` 阅读态与 `/share/{token}` 显示一致
- 图片、链接卡片、评论高亮只读渲染一致

## 15. 结论

这条需求最关键的实现原则只有两条：

1. 权限判断统一在后端收口
2. 分享页必须复用现有文档查看组件，只通过能力开关锁死编辑能力

按这个方向做，既能满足你的权限/分享需求，也不会重新引入第二套阅读页实现导致的样式和行为分叉。
