# CloudDoc 协作规则

## 文档同步

- 本项目以后所有需求文档、设计文档、测试计划、README、进度记录等文档类修订，都必须同步更新本地相关文档和在线 CloudDoc 云文档平台上的对应文档。
- 如果在线 CloudDoc 中还没有对应文档，需要先创建对应文档，再写入或同步本地文档内容。
- 在线 CloudDoc 同步必须使用已有顶层 `clouddoc` 文件夹作为统一根目录，不再创建新的顶层文件夹。
- 如需分类整理在线文档，只能在 `clouddoc` 文件夹下面按需求创建分类子文件夹。
- 当前在线 CloudDoc 的 `clouddoc` 文件夹 ID 为 `6c5bdd27-95c3-426d-93dc-d9e03739ab9a`。
- 如果发现此前同步到 `newdoc` 或其他非 `clouddoc` 目录的项目文档，需要迁移到 `clouddoc` 下对应分类目录。
- 如果只是代码变更但影响用户使用、部署、权限、MCP、数据模型或测试流程，也需要同步更新相关本地文档和在线 CloudDoc 文档。
- 每次完成文档同步后，需要在 `DEVELOPMENT_PROGRESS.md` 记录同步范围和验证结果。
